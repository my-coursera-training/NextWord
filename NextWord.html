NextWord

Next word prediction model based on previous context

Overview

"Next Word" application is intended to fulfil Coursera Data Science Specialization Capstone Project.

The training has been conducted on 80% of the available en_US "news" and "twitter" texts, that have been preprocessed to remove puntuation symbols and converted to lowercase.

After trying several packages (tm, quanteda,...), simple regular expresion hand made tokenizer has been user to extract process the texts, and package data.table has been used to create the tables unigrams, bigrams and trigrams frequency tables. In the last 2 tables, tokens strings have been replaced by the unigrams ID to save memory space.

The vocabulary has been trimmed to the tokens that represent 95% of associated words intances, associating the count of the trimmed token to "-NA-" token.

Lastly, bigram and trigrams tables have been processen to replacing tokens not present in vocabulary by "-NA-" token code (1) and have been limmited to lower of 95% of instance counts and a maximum of 4000000 rows.

Prediction Process

The main focus of the application has been to try to get it working usin a limited amount of memory and getting a response time as quick as posible.

Unique Size (MB)
Unigrams 15711 1.12

After the research on the subject, several techniques have been tested (some in a limited way due to lack of time), and finnally the simple "stupid" backoff approach was selected to be implemented in the application, as the benefits of more complex processes have been negligible (in the tests conducted).

Application Usage

The user can introduce a sentence in the left panel and press the button "predict" to get the prediction:

  • Number of alternatives: number of posible words to provide.

  • Predict missing values: Include the unknown word "-NA-" string as alternative when it is most most probable that the next wordis one not included in the Vocabulary. (the vocabulary can be reviewed in the "vocabulary" tab)

On the main panel, three tabs allow the user the usage information (previous to entering any word in the input box):

  • Prediction: shows the prediction of the next word depending on the configuration parameters.

  • Memory usage: The file with the compresed data of the tables that contains the unigrams, bigrams and trigrams in ".Rdata" format has a size of 12 MB. In this tab the user can see the actual size of each object in memory.

  • Vocabulary. As stated in the overview slide, the vocabulary has been restricted to thos tokens representing 95% of word instances of the taining set. In this tab the user can ispect the selected features.
